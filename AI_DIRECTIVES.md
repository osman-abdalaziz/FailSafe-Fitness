1. البروتوكول الإداري الصارم (Management Protocol)

    تعريف الأدوار: المستخدم (أنت) هو "مدير المشروع" وصاحب القرار النهائي والمنفذ المادي. أنا (Gemini) "المبرمج الرئيسي" المسؤول عن كتابة الكود النظيف واكتشاف الأخطاء.

    قاعدة التفتيش (GitHub First): يمنع كتابة أو تعديل أي كود قبل طلب رابط مستودع GitHub لمراجعة آخر تحديث (Commit) لضمان التوافق.

    التعديل الجراحي (Modular Updates): عند طلب تعديل، يتم استهداف الجزء المطلوب فقط دون العبث بالكود المستقر.

    بروتوكول الاختبار (Testing Protocol): كل كود جديد يجب أن يرفق معه خطوات ميكانيكية لكيفية اختباره محلياً من قبل المدير.

2. الرؤية الفنية والميزات (Project Vision & Features)

تطبيق ويب (PWA) فائق السرعة، مبني بـ (Vanilla JS, HTML5, CSS3) مع (Firebase)، يهدف لتتبع التدريب الميكانيكي والتغذية الدقيقة.
أ. وحدة التدريب (Advanced Hevy Clone)

    مرونة كاملة: إضافة أي عدد من المجموعات (Sets) وتصنيفها (تسخين، أساسية، فشل).

    Ghost Placeholder: عرض أوزان وعدات الجلسة السابقة كخلفية شفافة لتوجيه الزيادة التدريجية.

    كتالوج التمارين: استدعاء محلي من wger API للسرعة المطلقة.

ب. وحدة التغذية (AI Nutrition Router)

    إدخال ثلاثي: (نصي ذكي، بصري عبر الكاميرا، يدوي/باركود).

    الجدول التفاعلي: عرض النتائج في جدول قابل للتعديل اليدوي قبل الحفظ.

    توجيه السعرات (Borrowing): خصم تجاوز السعرات من الأيام القادمة مع حد أدنى (Hard Cap) لضمان الأداء السريري والتركيز الدراسي.

    رادار الميكرونوتريت: تنبيهات نقص الألياف والإلكتروليتات (بوتاسيوم/صوديوم).

ج. المحركات السريرية (AI Engines)

    التكرار المتباعد الحركي: تحليل الملاحظات الحركية عبر AI وتحويلها لتنبيهات "تصحيح مسار" تظهر قبل التمرين القادم.

    التنظيم التلقائي (Auto-Regulation): فحص جودة النوم والإجهاد العصبي (CNS) قبل الجلسة.

    التنبؤ بالهضبة (Plateau Prediction): اكتشاف ثبات الوزن والأداء قبل وقوعه وتقديم تقرير تشخيصي.

د. القياسات والمشتريات

    كاميرا التتبع المعياري (Ghost-Overlay): وضع صورة الأسبوع الماضي كطبقة شفافة فوق الكاميرا لتوحيد زاوية التصوير.

    محرك البقالة الذكي: توليد قائمة مشتريات دقيقة بناءً على الوجبات الناجحة في الأسبوع السابق.

    المراجعة الأسبوعية: قفل تحديثات الأهداف حتى إدخال قياسات يوم الجمعة (الوزن والخصر).

3. المكدس التقني (Tech Stack)

    Frontend: HTML5, CSS3, Vanilla JavaScript (No Frameworks).

    Backend: Firebase (Firestore, Cloud Functions).

    AI Core: Gemini 1.5 Pro (BYOK System).

    External APIs: wger API, Open Food Facts API, Google Fit REST API.

    المرحلة: التأسيس (Initial Setup).
