# AI_DIRECTIVES.md - FailSafe Fitness Project Constitution

## 1. Strict Management Protocol

- **Role Definition:** The User is the "Project Manager" (PM), final decision-maker, and physical implementer. I (Gemini) am the "Lead Developer" responsible for writing clean code and debugging.
- **GitHub First Rule:** No code will be written or modified before requesting the GitHub repository link to review the latest commit, ensuring compatibility.
- **Surgical Updates (Modular):** When a modification is requested, only the specific target component will be altered. Stable code must never be touched to prevent regressions.
- **QA & Testing Protocol:** Every new piece of code must be accompanied by clear, mechanical steps on how the PM can test it locally.

## 2. Project Vision & Features

An ultra-fast Progressive Web App (PWA) built with Vanilla JS, HTML5, CSS3, and Firebase, designed for strict mechanical training tracking and precise clinical nutrition management.

### A. Mechanical Training Unit (Advanced Hevy Clone)

- **Absolute Flexibility:** Add any number of sets per exercise and classify them (Warm-up, Normal, Drop-set, Failure).
- **Ghost Placeholder:** Display the previous session's weight and reps as a transparent background placeholder in input fields to visually enforce Progressive Overload.
- **Smart Catalog:** Local caching of exercises fetched from the wger API for zero-latency searching.

### B. Hybrid Nutrition Unit (AI Nutrition Router)

- **Tri-Channel Entry (Zero-Friction):** Smart NLP text input, Visual (AI Camera estimation), and Manual/Barcode scanning.
- **Interactive DOM Table:** All AI outputs render into an editable JavaScript table for manual verification before saving.
- **Calorie Borrowing:** Automatically deduct over-consumed calories from upcoming days with a "Hard Cap" minimum to protect cognitive and clinical performance.
- **Micronutrient Radar:** AI analysis to trigger warnings for severe deficiencies in fiber or electrolytes (Potassium/Sodium).

### C. AI Clinical Engines

- **Biomechanical Spaced Repetition:** AI analyzes post-workout form notes and generates corrective motor cues that appear as mandatory pop-ups before the same exercise in the next session.
- **Auto-Regulation:** Pre-workout CNS and sleep quality check to suggest volume adjustments if under-recovered.
- **Plateau Prediction Engine:** Mathematical algorithm monitoring waist circumference and gym weights to detect flattening progress curves early and suggest macro adjustments.

### D. Metrics & Grocery Auditing

- **Ghost-Overlay Camera:** Displays last week's progress photo as a transparent overlay on the camera viewfinder to ensure perfect alignment, lighting, and angle consistency.
- **AI Grocery Engine:** Analyzes successful macro-compliant days from the past week to generate a precise, gram-by-gram grocery list.
- **Weekly Clinical Audit:** Friday lockdown; weekly macro targets will not update until the mandatory weight and waist measurements are logged.

## 3. Tech Stack

- **Frontend:** HTML5, CSS3, Vanilla JavaScript (No Frameworks).
- **Backend:** Firebase (Firestore, Cloud Functions).
- **AI Core:** Gemini 1.5 Pro (BYOK - Bring Your Own Key).
- **External APIs:** wger API, Open Food Facts API, Google Fit REST API.
